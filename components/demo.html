Safety Companion Components

A collection of custom web components built with Material Web and RGA design tokens. These components follow Material Design 3 principles and use the Safety Companion design system.

sc-top-navbar

Primary, fixed top navigation element used across the app. It anchors global navigation controls and remains visible as users move through the experience.

Default State

<sc-top-navbar></sc-top-navbar>

With Notification Badge (Red Dot)

<sc-top-navbar notification-count="true"></sc-top-navbar>

Without Logo

<sc-top-navbar show-logo="false" notification-count="true"></sc-top-navbar>

Interactive Demo

Event Log:

API Reference

Attribute Type Default Description
show-logo boolean true Show or hide the centered logo
show-notification boolean true Show or hide the notification button
notification-count string "" Shows red dot badge when set to any truthy value

Events

Event Description
menu-click Fired when the menu button is clicked
notification-click Fired when the notification button is clicked

sc-chat-bubble

Standard message container used in conversational flows. Mirrors the interaction and visual behavior of ChatGPT's chat bubbles with Send (user) and Reply (assistant) variants.

Send Variant (User Message)

<sc-chat-bubble chat="Send" message="Hello! How can I help you today?"></sc-chat-bubble>

Reply Variant (Assistant Message)

<sc-chat-bubble chat="Reply" message="I'm here to help! What would you like to know?"></sc-chat-bubble>

Conversation Example

Multi-line Content

API Reference

Attribute Type Default Description
chat string "Send" Variant type: "Send" (user) or "Reply" (assistant)
message string "Text input" Message content to display in the bubble

sc-chip

Compact, at-a-glance status indicator. Primarily used within cards to communicate high-level states such as "Critical," "Safe," "Completed," "In Progress," "Dismissed," and "Recommended."

All States

Without Icons

<sc-chip state="Safe" show-icon="false"></sc-chip>

Without Trailing Icon

<sc-chip state="Safe" show-trailing-icon="false"></sc-chip>

Custom Labels

<sc-chip state="Safe" label="All Clear"></sc-chip>

Interactive Demo

Event Log:

API Reference

Attribute Type Default Description
state string "Safe" State variant: "Safe", "In progress", "Resolved", "Recommended", "Critical", "Dismissed", "Completed"
label string state value Custom label text to display
show-icon boolean true Show or hide the leading icon
show-trailing-icon boolean true Show or hide the trailing close icon

Events

sc-label-carousel

Horizontal scrollable container of filter chips. Filter chips use tags or descriptive words to filter content. Tapping on a filter chip activates it and changes its appearance to show selection state.

Default

<sc-label-carousel></sc-label-carousel>

Custom Labels (Array)

<sc-label-carousel labels='["All", "Security", "Privacy", "Updates", "Alerts"]'></sc-label-carousel>

Custom Labels (Comma-separated)

<sc-label-carousel labels="Home, Work, Personal, Family, Friends"></sc-label-carousel>

Interactive Demo

Selection Log:

API Reference

Attribute Type Default Description
labels string | array 8 "Label" items JSON array or comma-separated string of label texts

Events

Event Description Detail
selection-change Fired when chip selection changes { selected: number[], labels: string[] }

Visual States

  • Deselected: Transparent background with outline-variant border
  • Selected: Tertiary-container background, no border
  • Hover: Slight opacity change
  • Active: Scale down effect on click

sc-progress-indicator

Loading animation used to show active processing. Behaves similarly to ChatGPT typing/loading indicator. Appears inline within the conversation flow to communicate that a response is actively being generated.

Progress Values

Without Logo

<sc-progress-indicator value="60" show-logo="false"></sc-progress-indicator>

Without Thinking Bubbles

<sc-progress-indicator value="40" show-loading-bubbles="false"></sc-progress-indicator>

Custom Loading Text

<sc-progress-indicator value="60" loading-text="Processing your request..."></sc-progress-indicator>

Animated Demo

API Reference

Attribute Type Default Description
value number 0 Progress value (0, 20, 40, 60, 80, 100)
loading-text string "On it! Just a moment..." Text to display below the progress indicator
show-loading-bubbles boolean true Show or hide animated thinking bubbles
show-loading-text boolean true Show or hide loading text
show-logo boolean true Show or hide McAfee logo in center

Features

  • Circular progress: Red progress ring that animates based on value
  • Thinking bubbles: Animated pulsing dots that indicate active processing
  • McAfee logo: Centered brand icon within the progress circle
  • Loading text: Customizable message below the indicator
  • Smooth transitions: Progress updates animate smoothly

Dedicated text element used for greetings or welcome messaging. Appears only on the homepage or welcome screen, positioned near the top to anchor the page context. Non-interactive and typographically distinct.

Different Greetings

Custom Name

API Reference

Attribute Type Default Description
greeting string "Good afternoon" Greeting text (e.g., "Good morning", "Welcome back")

Typography

  • Letter spacing: -0.5px

  • Name: Medium weight (500)
  • Color: on-surface (#1C1B1D)

sc-side-sheet

Sliding sidebar navigation that adapts to both mobile and desktop layouts. Contains navigation menu items, CTAs, and supporting actions. On mobile, it slides in from the edge; on desktop, it can remain static.

Default

<sc-side-sheet></sc-side-sheet>

Custom Title

<sc-side-sheet title="Navigation Menu"></sc-side-sheet>

Without Actions

<sc-side-sheet show-actions="false"></sc-side-sheet>

Interactive Demo

Event Log:

API Reference

Attribute Type Default Description
title string "Title" Header title text
show-back boolean true Show or hide back button
show-close boolean true Show or hide close button
show-logo boolean true Show or hide McAfee logo
show-actions boolean true Show or hide action buttons (Save/Cancel)
show-footer boolean true Show or hide footer with sign out

Events

Event Description Detail
back Fired when back button is clicked -
close Fired when close button is clicked -

Features

  • Header: Back button, McAfee logo, title, and close button
  • Menu Items: 10 default navigation items with icons
  • Action Buttons: Save (filled) and Cancel (outlined)
  • Footer: Sign out option with icons
  • Dividers: Separate sections visually
  • Scrollable Content: Menu items scroll if content exceeds height

sc-horizontal-card

Structured card layout that surfaces issues, statuses, and system prompts. Contains issue title, timestamp, status chip, and details link. Cards flex horizontally, presenting all key information in a concise, scannable row.

All States

Different Status Chips

Custom Content

<sc-horizontal-card header="Security Alert Detected" subhead="2 minutes ago" chip-state="Critical"></sc-horizontal-card>

Interactive Demo

Event Log:

API Reference

Attribute Type Default Description
state string "outlined" Card state: "outlined", "elevated", or "focused"
header string "Header" Main title text
subhead string "Subhead" Timestamp or subtitle text
link string "Link" Link text (e.g., "Show details")
description string "description" Additional description text
status string "On" Status text displayed on right
chip-state string "Safe" Chip state (passed to sc-chip)
chip-label string "Safe" Chip label text

Events

Event Description
card-click Fired when the card is clicked

Features

  • Icon: Leading mood icon (Material Symbols)
  • Status Chip: Embedded sc-chip component for status display
  • Status Text: Right-aligned status indicator
  • Trailing Icon: Chevron right for navigation hint
  • Content: Header, subhead, link, and description text
  • States: Outlined (border), Elevated (shadow), Focused (background)

sc-bottom-text-input

Primary entry field for sending prompts. Functionally identical to ChatGPT bottom input bar. Anchored at the bottom of the screen with attach, text input, and send/record buttons.

Default State

<sc-bottom-text-input></sc-bottom-text-input>

Custom Placeholder

<sc-bottom-text-input placeholder="Ask me anything..."></sc-bottom-text-input>

Interactive Demo

Event Log:

API Reference

Attribute Type Default Description
placeholder string "Input text" Placeholder text for the input field
value string "" Initial value of the input field

Events

Event Description Detail
attach Fired when attach button is clicked -
send Fired when send button is clicked or Enter is pressed { value: string }
record Fired when record button is clicked -
input-change Fired when input value changes { value: string }

Features

  • Attach Button: Plus icon for attaching images or files
  • Text Input: Expandable textarea that grows with content
  • Send Button: Arrow up icon, active when text is entered
  • Record Button: Microphone icon for voice input
  • Auto-expand: Height increases from 56px to 129px when typing
  • Enter to Send: Press Enter to send (Shift+Enter for new line)
  • State Management: Automatically switches between default, input, and record states
